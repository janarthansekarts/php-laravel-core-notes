# PHP & Laravel Core Notes

This repository contains my personal learning notes from deeply studying **PHP Object-Oriented Programming** and **Laravel framework internals**.

These are **not tutorials or documentation summaries**.  
They are written to build **conceptual clarity** around *why things exist* and *how they work internally*, from the perspective of a working software engineer who wants a strong mental model of the framework and the language beneath it.

The focus of these notes is on:
- PHP OOP fundamentals (from first principles)
- How Laravel is designed and structured internally
- The architectural problems frameworks like Laravel are trying to solve
- Connecting language-level concepts to real-world framework behavior

---

## Who this repository is for

- **Me (primary)**: a long-term reference and second brain  
- Other developers who want a deeper understanding beyond syntax and surface-level usage  

If you’re looking for quick start guides, CRUD tutorials, or “how to build X in 5 minutes,” this repository is **not** intended for that purpose.

---

## How the notes are organized

The content is structured **progressively**, in the same order that the concepts build on each other conceptually.

Each section focuses on a specific layer of understanding:
- Core PHP Object-Oriented concepts
- Laravel’s big-picture architecture
- Framework bootstrapping and request lifecycle
- Internal behavior of Eloquent and database interactions
- Advanced framework concepts and design decisions

Each Markdown file focuses on **one concept at a time**, keeping scope intentionally narrow so ideas are easier to reason about and connect.

---

## Writing principles I follow

- Explain **WHY before HOW**
- Every concept is described before it is used
- Start with simple analogies, then move to technical detail
- Avoid surface-level summaries and copied documentation
- Prefer clarity and correctness over brevity

These notes are meant to reflect understanding, not repetition.

This repository is actively updated as I study, revisit concepts, and refine my understanding.
Notes may change over time as my mental model becomes clearer and more precise.
